# 02 — 信息架构与界面交互

> 描述当前已实现的主界面结构（随产品演进更新）。

## 1. 设计目标

一屏完成主任务：**发现技能 → 看懂技能 → 选中目标项目 → 复制 / 新建项目落地**。  
轻量：无仪表盘堆砌；首屏即工作台。术语面向所有用户（少用 Runtime / Registry 等 jargon）。

## 2. 导航结构

```
App
├── 技能库（默认）          # 两栏工作台 + 详情二级窗
├── 组合包                  # 常用技能组合
├── 健康检查                # 健康报告列表
├── 新建项目                # 目录脚手架 + 选技能 + 终端安装 + 健康检查
├── 在线安装                # 打开终端执行 npx skills（全局）
├── 设置                    # 策略模板、ZIP 导入等
├── 来源与项目              # 扫描源开关、项目根登记
└── 操作记录                # OpLog
```

以上入口均已可用。

## 3. 技能库布局

```text
┌──────────────┬──────────────────────────────────────────────┐
│ Left Rail    │ Catalog                                      │
│ ~232px       │                                              │
├──────────────┼──────────────────────────────────────────────┤
│ 搜索 (/)     │ 工具条：计数 / 多选 / 复制 / ZIP …           │
│ 适用工具     │ 目标项目条（路径 · 工具 · 冲突 · 拖放）       │
│ 副本 / 收藏  │ 操作预览（有复制预览时展开）                 │
│ 标签筛选     │ 技能卡片网格                                 │
│ 常用入口     │                                              │
│ 组合包短列表 │                                              │
└──────────────┴──────────────────────────────────────────────┘

点击卡片 → 右侧滑出二级详情窗（独立滚动；Esc / 点遮罩关闭）
```

### 3.1 左栏

| 区块 | 内容 | 交互 |
|------|------|------|
| 搜索 | 名称 / 说明 | `/` 聚焦；过滤中栏 |
| 适用工具 | Cursor / Claude / Agents / … | 多选筛选 |
| 开关 | 仅看有副本、仅看收藏 | |
| 标签 | 用户标签列表 | 单选筛选；详情内增删 |
| 常用 | 最近使用 / 收藏快捷入口 | 点选打开详情窗 |
| 组合包 | 短列表 + 用已选创建 | 进入组合包页 / 创建 |

### 3.2 中栏（Catalog）

**顶部目标项目条：**

| 元素 | 说明 |
|------|------|
| 当前项目 | 路径；未选择时提示 |
| 选择项目 / 新建项目 | 浏览文件夹；跳转新建项目页 |
| 复制到哪些工具 | Agents / Claude / Cursor 多选 |
| 文件已存在时 | 覆盖 / 跳过 / 另存新名称 |
| 拖放区 | 卡片拖入生成复制预览；确认复制 / 取消 |

**卡片字段（列表不加载全文）：**

- 名称、截断说明
- 徽章：适用工具、scope、只读、有脚本、副本、标签、健康分

**交互：**

| 操作 | 行为 |
|------|------|
| 单击 | 选中并打开详情二级窗 |
| 多选模式 / Ctrl/⌘ / Shift | 勾选，不打开详情 |
| 拖到目标项目条 | 生成预览 → 确认复制 |
| 工具条 | 复制到项目、导出 ZIP、收藏、删除 |

### 3.3 详情二级窗

从右侧滑出；标题栏固定，正文区独立滚动。内容包括：

1. **标题区**：名称、路径、徽章、打开目录 / 提取为自己的副本 / 排版·原文切换 / 关闭  
2. **导语**：description 全文  
3. **标签**：增删（扫描时保留）  
4. **健康**：有问题时展示摘要；完整报告见「健康检查」页  
5. **副本**：一致 / 有差异；查看 diff；以当前为准同步  
6. **脚本风险 / 变更历史**：有则显示；历史默认折叠  
7. **附属文件**：chip 列表  
8. **正文**：大纲可折叠；友好 Markdown 或原文视图  

不做内嵌可视化编辑器；「打开目录」用资源管理器处理文件。

## 4. 新建项目页

单页流水线（非强制步进器，可上下滚动操作）：

1. **项目目录** — 选择 / 打开；可选「分析技术栈并推荐」并将匹配技能加入已选  
2. **工具目录** — 默认勾选 `.claude` / `.agents` / `.cursor`；冲突策略  
3. **本机技能** — 搜索多选  
4. **skills.sh** — 打开交互终端（`cwd`=项目），预执行 `npx skills add <包>` 等基础命令  
5. **一键落地** — `scaffold` 建目录（含 `.gitkeep`）→ 复制已选 → 项目范围健康检查  
6. **健康结果** — 列出本项目技能；标出可能非最新  

后端能力：`scaffold_project`、`open_skills_cli_terminal`、`run_health_scan_scoped`、既有 `preview_copy` / `execute_copy`。

## 5. 在线安装页

- **交互类**（find / add / update / remove）：打开系统终端，预填**最简**命令（不预填 `--yes` / `--copy` / `-a` / `-s`）  
- **已安装列表**：应用内捕获 `npx skills list -g` 输出  
- 全局安装命令带 `-g`；新建项目页不带 `-g` 且 `cwd` 为项目根  

## 6. 关键用户流程

### F1 — 搞清本机有哪些技能

1. 启动加载索引；可点「重新扫描」  
2. 左栏按适用工具 / 副本 / 标签过滤  
3. 点开卡片看详情  

### F2 — 新建项目落地

1. 打开 **新建项目**  
2. 选目录 → 勾选工具文件夹 → 选本机技能和/或终端安装  
3. 一键落地 + 健康检查  
4. **操作记录** 可追溯  

### F3 — 从只读插件提取自用

1. 打开只读技能详情  
2. 「提取为自己的副本」→ 写入 `~\.agents\skills`  

### F4 — 清理重复副本

1. 勾选「仅看有副本」  
2. 详情中 Diff → 同步或删除多余（仅可写源）  

## 7. 其它页面（摘要）

| 页 | 能力 |
|----|------|
| 组合包 | 列表、创建、导入导出 JSON、应用到项目、删除 |
| 健康检查 | 报告列表、展开问题、打开目录、重新检查；skills.sh 对照 diff |
| 新建项目 | 见 §4 |
| 在线安装 | 见 §5 |
| 设置 | 策略模板、ZIP 导入、插件拷贝限制 |
| 来源与项目 | 启用源、登记项目根 |
| 操作记录 | 近期写操作 |

## 8. 交互与安全

| 规则 | 说明 |
|------|------|
| 拖拽默认复制 | 生成预览后确认 |
| 删除二次确认 | 展示副本数、组合包引用；进回收站 |
| 只读源 | 禁止就地删除/覆盖；可提取副本 |
| skills.sh | 选项在系统终端完成，应用不替用户静默选定 |
| 键盘 | `/` 搜索；`Del` 删除（确认）；`Esc` 关 diff → 详情 → 状态条 |

## 9. 视觉与轻量约束

- 单窗口；无强制托盘常驻  
- 详情按需读盘；列表只绑索引  
- 不内嵌完整 IDE / Agent 对话
