# 使用指南

面向最终用户的简要操作说明。开发者请先阅读仓库 [README](../README.md)。

## 1. 第一次打开

1. 安装并启动 SSM（Windows x64）
2. 点击顶部 **重新扫描**，等待索引本机 skill 源
3. 若项目级 skill 看不到：打开 **源与项目**，登记项目根目录后再扫描

## 2. 浏览与搜索

- 中间为技能卡片；左侧可按适用工具、副本、收藏、标签筛选
- 按 `/` 聚焦搜索框（匹配名称 / 说明）
- 单击卡片：弹出详情二级窗（独立滚动；`Esc` 关闭）
- 界面上的 **?** 可悬停查看简短说明

## 3. 复制到项目

1. 在技能库顶部 **目标项目** 条中 **选择项目**（或先在「来源与项目」登记）
2. 勾选要复制到哪些工具（Cursor / Claude / Agents）
3. 将卡片 **拖入** 目标项目条，或工具条 **复制到项目**
4. 核对预览路径后点 **确认复制**

「文件已存在时」的处理方式可在目标项目条或 **设置** 中调整。

## 4. 副本与同步

同名 skill 出现在多处时，卡片会标 `twins`：

1. 打开详情 → **副本**
2. **diff** 查看 `SKILL.md` 差异
3. 确认后可用 **同步**（以当前为准覆盖另一侧；只读目标不可覆盖）

## 5. 组合包 Bundle

1. 在技能库多选 skill，填写名称创建组合包；或在 **组合包** 页管理
2. 选好目标项目后，对 Bundle **应用到目标项目**
3. 也可在 **向导** 页：分析项目技术栈 → 推荐并生成 Bundle

支持 Bundle JSON 导入 / 导出，便于分享组合。

## 6. 健康检查

1. 打开 **健康** 页，可 **重新检查全部**
2. 展开卡片查看问题；可 **打开目录** 到资源管理器
3. 部分问题（如缺少 name）支持 **应用修复**
4. 仅通过 `npx skills` 安装、且写入锁文件的 skill，才会对照 skills.sh 远端版本并显示 diff

本地手写的 skill **不会**因为「不在锁文件里」被扣分。

## 7. 在线安装（skills.sh）

在 **在线安装** 页可搜索、列出、安装、更新、移除技能（底层调用本机 `npx skills`）。

需本机已安装 Node.js，且网络可访问相应源。安装后请点「重新扫描」。

## 8. 导入导出与其它

| 能力 | 入口 |
|------|------|
| 选中技能打 ZIP | 技能库工具条 |
| 导入 ZIP | 设置页 |
| 只读源提取自用 | 详情「提取为自己的副本」→ 通常写入 `~\.agents\skills` |
| 删除 | 工具条或 `Del`（确认后进回收站） |
| 操作追溯 | **操作记录** 页 |

## 9. 快捷键

| 键 | 作用 |
|----|------|
| `/` | 搜索 |
| `Del` | 删除选中（仍确认） |
| `Esc` | 依次关闭：副本 diff → 详情窗 → 状态条 |
