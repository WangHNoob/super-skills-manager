# 使用指南

面向最终用户的简要操作说明。开发者请先阅读仓库 [README](../README.md)。

## 1. 第一次打开

1. 安装并启动 SSM（Windows x64）
2. 点击顶部 **重新扫描**，等待索引本机 skill 源
3. 若项目级技能看不到：打开 **来源与项目**，登记项目根目录后再扫描

## 2. 浏览与搜索

- 中间为技能卡片；左侧可按适用工具、副本、收藏、标签筛选
- 按 `/` 聚焦搜索框（匹配名称 / 说明）
- 单击卡片：弹出详情二级窗（独立滚动；`Esc` 关闭）
- 界面上的 **?** 可悬停查看简短说明

## 3. 复制到项目

1. 在技能库顶部 **目标项目** 条中 **选择项目**（或先在「来源与项目」登记）
2. 勾选要复制到哪些工具（Cursor / Claude / Agents）
3. 将卡片 **拖入** 目标项目条，或工具条 **复制到项目**
4. 核对预览路径后点 **确认复制**

「文件已存在时」的处理方式可在目标项目条或 **设置** 中调整。

## 4. 副本与同步

同名技能出现在多处时，卡片会标「副本」：

1. 打开详情 → **副本**
2. **diff** 查看 `SKILL.md` 差异
3. 确认后可用 **同步**（以当前为准覆盖另一侧；只读目标不可覆盖）

## 5. 新建项目（推荐）

在 **新建项目** 页可一次完成：

1. 选择项目文件夹  
2. 勾选需要的工具目录（`.claude` / `.agents` / `.cursor`，默认全选）  
3. 从本机技能库多选，或按技术栈推荐加入  
4. 从 skills.sh **安装到本项目**（非全局）  
5. **创建目录并复制已选技能**  
6. 对本项目做健康检查，标出可能非最新的技能  

## 6. 组合包

1. 在技能库多选技能，填写名称创建组合包；或在 **组合包** 页管理  
2. 选好目标项目后 **应用到目标项目**  
3. 支持组合包 JSON 导入 / 导出，便于分享

## 7. 健康检查

1. 打开 **健康检查** 页，可 **重新检查全部**
2. 展开卡片查看问题；可 **打开目录** 到资源管理器
3. 部分问题（如缺少 name）支持 **应用修复**
4. 仅通过 `npx skills` 安装、且写入锁文件的 skill，才会对照 skills.sh 远端版本并显示 diff

本地手写的 skill **不会**因为「不在锁文件里」被扣分。

## 8. 在线安装（skills.sh）

搜索 / 安装 / 更新 / 移除会**打开系统终端**，并自动执行基础 `npx skills …` 命令；需要选择技能或目标工具时，在终端里按提示操作。

- **新建项目** 页：终端工作目录为当前项目（项目级安装）
- **在线安装** 页：默认全局（命令带 `-g`）

「已安装列表」仍在应用内直接显示。需本机 Node.js；终端操作完成后请点「重新扫描」。

## 9. 导入导出与其它

| 能力 | 入口 |
|------|------|
| 选中技能打 ZIP | 技能库工具条 |
| 导入 ZIP | 设置页 |
| 只读源提取自用 | 详情「提取为自己的副本」→ 通常写入 `~\.agents\skills` |
| 删除 | 工具条或 `Del`（确认后进回收站） |
| 操作追溯 | **操作记录** 页 |

## 10. 快捷键

| 键 | 作用 |
|----|------|
| `/` | 搜索 |
| `Del` | 删除选中（仍确认） |
| `Esc` | 依次关闭：副本 diff → 详情窗 → 状态条 |
